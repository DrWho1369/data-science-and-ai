🔹 What is Object-Oriented Programming (OOP)?

OOP is a programming paradigm that models real-world concepts using objects. These objects contain both data (attributes) and behaviors (methods).

✅ Instead of writing functions separately, OOP bundles data and behavior together, making the program more organized.

🔹 Defining a Class

A class is a blueprint for creating objects. Think of it like a cookie cutter—it defines the shape, but the actual cookies (objects) are made from it.

class Person:
    """Defines a person with a name and age."""
    def __init__(self, name, age):
        self.name = name  # Instance variable for storing name
        self.age = age  # Instance variable for storing age

# Creating an object (instance of the class)
person1 = Person("Alice", 30)

# Accessing attributes
print(person1.name)  # Output: Alice
print(person1.age)   # Output: 30

✅ The self keyword refers to the current instance of the class. It allows the object to store data inside itself.

🔹 Attributes & Instance Variables

Every object has its own data. Instance variables store this data:

class Car:
    def __init__(self, brand, model):
        self.brand = brand  # Each object will have its own brand
        self.model = model  # Each object will have its own model

car1 = Car("Toyota", "Corolla")
car2 = Car("Honda", "Civic")

print(car1.brand)  # Output: Toyota
print(car2.brand)  # Output: Honda

✅ car1 and car2 are separate objects, each with different data.

🔹 Encapsulation: Protecting Data

Encapsulation means hiding internal details of an object and only allowing controlled access.

✅ This prevents accidental modifications of sensitive data.

class BankAccount:
    def __init__(self, balance):
        self.__balance = balance  # Private variable (indicated by double underscores)

    def deposit(self, amount):
        """Adds money to the account."""
        self.__balance += amount

    def get_balance(self):
        """Returns the account balance."""
        return self.__balance

account = BankAccount(1000)
account.deposit(500)
print(account.get_balance())  # Output: 1500

✅ The double underscore __balance makes it private, meaning it can't be directly accessed outside the class.

🔹 Operator Overloading: Customizing Operators

OOP allows you to override built-in operators (e.g., +, -) so they work with objects.

class Time:
    def __init__(self, hours, minutes):
        self.hours = hours
        self.minutes = minutes

    def __add__(self, other):
        """Defines how + works for Time objects"""
        total_minutes = self.minutes + other.minutes
        total_hours = self.hours + other.hours + (total_minutes // 60)
        return Time(total_hours, total_minutes % 60)

time1 = Time(2, 50)
time2 = Time(1, 20)
time3 = time1 + time2  # Uses __add__
print(time3.hours, time3.minutes)  # Output: 4 10

✅ The __add__ method modifies how "+" behaves for objects of this class.

🔹 Type Annotations for Classes

Type annotations make it clear what type of data each variable should store.

class Student:
    name: str
    age: int

    def __init__(self, name: str, age: int):
        self.name = name
        self.age = age

✅ This helps avoid errors and makes the code more readable.

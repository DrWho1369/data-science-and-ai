🔹 Working with Files

Programs often need to read from or write to files for data storage.

# Open a file in write mode ("w") and store text
with open("example.txt", "w") as file:
    file.write("Hello, world!")  # Writes this text into the file

# Open the file in read mode ("r") and print its content
with open("example.txt", "r") as file:
    content = file.read()  # Reads the entire file
    print(content)  # Output: Hello, world!

✅ This code writes text to a file and then reads it back.

🔹 Modules & Libraries

A module is a file containing reusable functions and variables.

# Importing the math module to use mathematical functions
import math
print(math.sqrt(16))  # Output: 4.0 (Square root of 16)

✅ This allows us to use pre-built functions instead of writing them ourselves.

🔹 Handling Exceptions

Exceptions occur when an error happens during execution. Instead of crashing, we can handle the error gracefully.

try:
    result = 10 / 0  # This will cause a ZeroDivisionError
except ZeroDivisionError:
    print("Cannot divide by zero!")  # This message will be printed instead of crashing the program

✅ The `try` block attempts code, and if an error occurs, the `except` block handles it instead of stopping the program.

🔹 Type Hints

Type hints help clarify what types of data functions and variables should use.

def greet(name: str) -> str:
    return "Hello, " + name

print(greet("Alice"))  # Output: Hello, Alice

✅ Here, `name: str` tells Python that `name` should be a string, and `-> str` means the function returns a string.

💻 Key Code / Functions

🔹 Opening & Writing to Files

Use the `with open()` method to ensure proper file handling.

with open("data.txt", "w") as f:
    f.write("Hello, world!")  # Saves text to the file

🔹 Importing & Using Modules

Python comes with built-in modules that help with various tasks.

import random
print(random.randint(1, 10))  # Generates a random number between 1 and 10

🔹 Exception Handling

Always use exception handling when dealing with user input, files, or network connections.

try:
    value = int("abc")  # Raises ValueError because "abc" is not a number
except ValueError:
    print("Invalid number format!")  # The program continues running

🔹 Using Type Hints

Type hints make functions more readable and maintainable.

def add(x: int, y: int) -> int:
    return x + y

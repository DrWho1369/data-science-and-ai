üîπ What is Inheritance?

Inheritance allows a class to reuse code from another class, making it easier to manage and reduce repetition.

‚úÖ Think of it like family traits: A child (subclass) inherits characteristics from a parent (superclass).

üîπ Creating a Superclass (Parent Class)

A superclass is a general class that contains common attributes and methods.

class Animal:
            """Superclass representing general animals."""
            def __init__(self, name):
                self.name = name  # Stores the animal's name
        
            def speak(self):
                return "Some generic animal sound"
        
        # Creating an instance of the superclass
        animal = Animal("Generic Animal")
        print(animal.speak())  # Output: Some generic animal sound

‚úÖ This Animal class defines a general animal with a speak() method.

üîπ Creating a Subclass (Child Class)

A subclass is a more specific version of a superclass. It inherits attributes/methods but can also define its own.

class Dog(Animal):
            """Dog class inherits from Animal"""
            def speak(self):
                return "Woof!"  # Overrides the speak method
        
        dog = Dog("Buddy")
        print(dog.name)  # Output: Buddy
        print(dog.speak())  # Output: Woof!

‚úÖ The Dog class inherits from Animal, but overrides speak() to return "Woof!".

üîπ Using the Super() Function

The super() function allows a subclass to call methods from its superclass.

class Cat(Animal):
            def __init__(self, name, color):
                super().__init__(name)  # Calls Animal's __init__
                self.color = color  # Adds a new attribute
        
            def speak(self):
                return "Meow!"
        
        cat = Cat("Whiskers", "Black")
        print(cat.name, cat.color)  # Output: Whiskers Black
        print(cat.speak())  # Output: Meow!

‚úÖ The super().__init__(name) line calls the constructor of the superclass (Animal), so we don‚Äôt have to rewrite it.

üîπ Method Overriding

A subclass can replace a method from its superclass with its own version. This is called overriding.

class Bird(Animal):
            """Bird class overrides speak() method"""
            def speak(self):
                return "Chirp chirp!"
        
        bird = Bird("Robin")
        print(bird.speak())  # Output: Chirp chirp!

‚úÖ Even though Bird inherited speak() from Animal, it overrides it to return "Chirp chirp!".

üîπ Polymorphism: Making Code More Flexible

Polymorphism allows us to use the same method on different types of objects.

# A list of different animals
        animals = [Dog("Rex"), Cat("Luna"), Bird("Tweety")]
        
        for animal in animals:
            print(animal.name, "says", animal.speak())
        
        # Output:
        # Rex says Woof!
        # Luna says Meow!
        # Tweety says Chirp chirp!

‚úÖ The same speak() method works for different classes. This is the power of polymorphism!

üîπ Multiple Inheritance

A class can inherit from multiple superclasses.

class Flyable:
            def fly(self):
                return "I can fly!"
        
        class Parrot(Animal, Flyable):
            """Parrot class inherits from both Animal and Flyable"""
            def speak(self):
                return "Squawk!"
        
        parrot = Parrot("Polly")
        print(parrot.speak())  # Output: Squawk!
        print(parrot.fly())    # Output: I can fly!

‚úÖ Parrot inherits from both Animal and Flyable, so it has both speak() and fly() methods.

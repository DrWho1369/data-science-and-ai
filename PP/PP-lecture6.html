🔹 What is Unit Testing?

Unit testing is a method of testing individual functions (small parts of a program) to ensure they work correctly.

✅ Helps catch bugs early and ensures that new changes don’t break existing code.

🔹 Writing Unit Tests in Python (`unittest`)

Python has a built-in module called `unittest` that allows us to test our code.

import unittest  # Import the testing framework

# Function to test
def add(x, y):
    return x + y

# Create a test case (a class that runs tests)
class TestMath(unittest.TestCase):
    def test_addition(self):
        # Check if add(2, 3) correctly returns 5
        self.assertEqual(add(2, 3), 5)  

# Run tests when the file is executed
if __name__ == "__main__":
    unittest.main()

✅ `assertEqual()` checks if the function returns the expected value. If not, the test fails.

✅ Running `unittest.main()` executes all test cases.

🔹 Test-Driven Development (TDD)

TDD is a software development approach where you write tests before writing code.

Steps in TDD:

  1. Write a failing test (because the function doesn't exist yet).
  2. Write just enough code to make the test pass.
  3. Refactor (improve) the code to make it cleaner.

🔹 Using Pytest for Testing

✅ `pytest` is a popular alternative to unittest, with a simpler syntax.

# Install pytest first: pip install pytest

# Function to test
def multiply(x, y):
    return x * y

# Test function
def test_multiply():
    assert multiply(2, 3) == 6  # ✅ Test passes

# Run with: pytest test_file.py

✅ `assert` is used instead of `self.assertEqual()` for simpler syntax.

🔹 Version Control: Why Is It Important?

Version control tracks changes to code, allowing developers to:

  • 🔹 Revert to previous versions if something breaks.
  • 🔹 Collaborate with multiple developers without overwriting each other’s work.
  • 🔹 Experiment with new features without affecting the main project.

🔹 Git Basics: Commands Every Developer Must Know

# ✅ Initialize a new Git repository
git init

# ✅ Check the status of files
git status

# ✅ Add files to be tracked
git add file.py

# ✅ Commit changes with a message
git commit -m "Added new feature"

# ✅ View commit history
git log

# ✅ Push changes to GitHub
git push origin main

# ✅ Pull latest changes from GitHub
git pull origin main

✅ Git tracks changes to code using commits.

🔹 GitHub: Hosting & Collaboration

GitHub is a cloud-based platform for hosting Git repositories, allowing developers to:

  • 🔹 Store and share code publicly or privately.
  • 🔹 Collaborate on projects with multiple contributors.
  • 🔹 Track issues and manage projects.

🔹 Working with GitHub: Pushing Code

# ✅ Connect local Git repo to GitHub
git remote add origin https://github.com/your-username/repo-name.git

# ✅ Push local changes to GitHub
git push -u origin main

✅ This command links a local project to GitHub and pushes changes.
